[#cpu]
== Adding CPU and Memory to a Running VM
このラボでは、仮想マシンの「インスタイプ（Instance Type）」を変更することにより、実行中の仮想マシンにCPUとメモリのリソースを追加します。まず、1つのvCPUと2 GiBのメモリを持つ既存のVMから開始します。OpenShift Virtualizationコンソールを使用して、VMのインスタンスタイプを 2 vCPU と 4 GiB のメモリを持つタイプに変更します。

[#cpuinstructions]
=== Instructions

. *Webブラウザ* から <<labaccess,admin ユーザーとして OpenShift コンソールにログイン>>していることを確認し、次のステップに進みます。

+
. *OpenShiftコンソール* を使用し、*Virtualization（仮想化） → Virtual Machines（仮想マシン）* に移動します。

+
*All projects（すべてのプロジェクト）* の下で、*hot-plug* ネームスペースを選択し、*hot-plug-vm1* VMを選択します。

+
InstanceType（インスタンスタイプ）が *u1.small* に設定されていることを確認します。

+
InstanceTypeの上部で、このVMの現在の *CPU | Memory* が *1 CPU | 2GiB Memory* になっていることが確認できます。

+
image::exercise3/03-image-cpu-001.png[title="CPU and Memory before hot plug", link=self, window=blank, width=100%]

+
. *ターミナル* ウィンドウから *virtctl* を使用して、*hot-plug-vm1* VMを起動します。

+
[,shell,role="execute wrap",subs="attributes"]
----
virtctl start hot-plug-vm1 -n hot-plug
----

+
[,shell,role=wrap,subs="attributes"]
.Output
----
VM hot-plug-vm1 was scheduled to start
----

+
. *Configuration（構成）* タブをクリックし、まだ選択されていない場合は *Details（詳細）* を選択します。
+
InstanceTypeが *General Purpose* で、*1 CPU | 2GiB Memory* になっていることが確認できます。

+
image::exercise3/03-image-cpu-002.png[title="Virtual Machine details", link=self, window=blank, width=100%]

. 青い鉛筆アイコンをクリックして、CPUとメモリのリソースを変更します。
+
表示されたウィンドウで、*Size（サイズ）* のドロップダウンをクリックし、*2xmedium: 2CPUs, 4Gi Memory* を選択します。
+
*Save（保存）* ボタンをクリックします。

+
image::exercise3/03-image-cpu-003.png[title="Edit Instance Type", link=self, window=blank, width=100%]

. Details（詳細）パネルの構成に、仮想マシンが2つのCPUと4 GiBのメモリを持つようになったことが表示されます。

+
image::exercise3/03-image-cpu-004.png[title="Virtual Machine details", link=self, window=blank, width=100%]

+
[NOTE]
====
新しいCPUとメモリの構成を有効にするために、仮想マシンはあるワーカーノードから別のワーカーノードへ自動的にマイグレーション（移行）されます。新しいCPUとメモリの構成がすぐに表示されない場合は、数分待つ必要があります。
====

+
image::exercise3/03-image-cpu-005.png[title="CPU and Memory messages", link=self, window=blank, width=100%]

. UIから *Console（コンソール）* タブをクリックします。必要に応じて、青い *Connect（接続）* ボタンをクリックします。
+
コンソールウィンドウの上部に表示されている *User name（ユーザー名）* と *Password（パスワード）* の資格情報を使用し、*Copy to clipboard（クリップボードにコピー）* と *Paste to console（コンソールに貼り付け）* を使って仮想マシンにログインします。

+
image::exercise3/03-image-cpu-006.png[title="Login to VM", link=self, window=blank, width=100%]

. VMに2つのCPUと4GiBのメモリがあることを検証するために、以下を実行します。

+
.プロセッサー数の取得
[source,text,role=execute]
----
nproc
----

+
.メモリ量の取得
[source,text,role=execute]
----
free -m
----
+
VMに2つのCPUと、合計で約4 GiBのメモリがあることが確認できます。

+
[NOTE]
====
このVMイメージでは kdump が有効になっており、カーネル用に一部のメモリが予約されています。そのため、メモリ量が4 GiB未満に見えます。実際には、VMには4 GiBのメモリが構成されていますが、オペレーティングシステムからはそれより低い数値として認識されます。kdump が無効であれば、合計メモリは4 GiBになります。
====

+
image::exercise3/03-image-cpu-007.png[title="Verify new resources", link=self, window=blank, width=100%]

[#disk]
== Adding a Virtual Disk to a Running VM

このラボでは、既存のVMに新しいディスクを追加します。1つ以上のディスクを持つ既存のVMから開始し、OpenShift Virtualizationコンソールを使用して、実行中にホットプラグされた追加ディスクが含まれるようにVMの構成を変更します。

[NOTE]
====
これは、VMwareの *Hot Add（ホットアッド）* 機能に類似しています。
====

[#diskinstructions]
=== Instructions

[start=1]
. *Webブラウザ* から <<labaccess,admin ユーザーとして OpenShift コンソールにログイン>>していることを確認し、次のステップに進みます。

+
. *OpenShiftコンソール* を使用し、*Virtualization（仮想化） → Virtual Machines（仮想マシン）* に移動します。

+
*All projects（すべてのプロジェクト）* の下で、*hot-plug* ネームスペースを選択し、*hot-plug-vm1* VMを選択します。

+
image::exercise3/03-image-disk-001.png[title="hot-plug-vm1 details", link=self, window=blank, width=100%]

+
. *Configuration（構成）* タブをクリックし、*Storage（ストレージ）* を選択します。仮想マシンには現在、*ocs-external-storagecluster-ceph-rbd* という名前の *Storage class（ストレージクラス）* を使用している30GBのブート可能ディスクが1つだけ存在していることが確認できます。

+
image::exercise3/03-image-disk-002.png[title="VM has only one disk", link=self, window=blank, width=100%]

. *Disks（ディスク）* の下にある *Add（追加）* ボタンをクリックし、*Empty disk (blank)（空のディスク（ブランク））* を選択して空のディスクを追加します。

+
image::exercise3/03-image-disk-003.png[title="Add an empty disk", link=self, window=blank, width=100%]

+
. 表示されたウィンドウで、新しいディスクの名前（例：*data-disk-01*）を入力し、サイズを 5 GiB に変更します。残りのオプションはデフォルトのままで構いません。
+
*Save（保存）* ボタンをクリックします。

+
image::exercise3/03-image-disk-004.png[title="Configure the new disk", link=self, window=blank, width=100%]

. 新しいディスクが *Persistent Hotplug（永続ホットプラグ）* のラベル付きで追加されました。

+
image::exercise3/03-image-disk-005.png[title="The disk is added", link=self, window=blank, width=100%]

+
. 上にスクロールして *Console（コンソール）* タブをクリックします。新しいディスクがアタッチされたというログ（*/dev/vdc*）がVNCコンソールの標準出力に表示されます。新しいディスクはすぐに表示されない場合があるため、しばらくお待ちください。
+
標準出力にログが表示されない場合は、`sudo dmesg` を実行することで、ディスクの追加とログを確認できます。

+
image::exercise3/03-image-disk-006.png[title="Kernel messages about a new disk", link=self, window=blank, width=100%]

+
. コンソールウィンドウの上部に表示されている *User name（ユーザー名）* と *Password（パスワード）* の資格情報を使用し、*Copy to clipboard（クリップボードにコピー）* と *Paste to console（コンソールに貼り付け）* を使って仮想マシンにログインします。必要に応じて、青い *Connect（接続）* ボタンをクリックします。
+
`lsblk` コマンドを実行すると、新しい 5 GiB のディスクが `vdc` として追加されていることが確認できます。
+
image::exercise3/03-image-disk-007.png[title="View the new disk from the OS", link=self, window=blank, width=100%]

. 新しいディスクは、ディスクへのアクセスを管理する新しいポッドを介して仮想マシンにホットプラグされます。この新しいポッドはVMと同じプロジェクト内に作成され、*hp-volume-xxxxx* という名前になります。このポッドは *virt-launcher* に加えて実行されます。

+
.ポッドリストの取得
[source,text,role=execute]
----
oc get pods -n hot-plug
----

+
.Output
----
NAME                                READY   STATUS      RESTARTS   AGE
hp-volume-mdnjd                     1/1     Running     0          114s
virt-launcher-hot-plug-vm1-rb6lg    2/2     Running     0          147m
----

+
. VMが停止すると *hp-volume* ポッドは停止し、VMが起動するとポッドは再び起動します。新しいディスクは通常のディスクと同様に使用でき、VMを停止して起動しても永続的に保持されます。

+
. ここでは行いませんが、この時点で新しいディスクにファイルシステムを作成し、マウントして必要に応じて使用することができます。

+
. 次に、ホットプラグしたディスクを仮想マシンから取り外します。

+
. *OpenShiftコンソール*を 使用し、*Virtualization（仮想化） → Virtual Machines（仮想マシン）* に移動します。
+
*All projects（すべてのプロジェクト）* の下で、*hot-plug* ネームスペースを選択し、*hot-plug-vm1* VMを選択します。
+
*Configuration（構成）* タブをクリックし、*Storage（ストレージ）* を選択します。
+
後でボーナスラボで使用する可能性があるため、削除するオブジェクトの名前（例：*dv-hot-plug-vm1-data-disk-01-um6O11*）を書き留めておいてください。
+
ホットプラグされたディスクの右側にある3つのドットをクリックし、*Detach（デタッチ）* を選択します。

+
image::exercise3/03-image-disk-011.png[title="Detach the disk", link=self, window=blank, width=100%]

+
. *Detach（デタッチ）* ボタンをクリックしてアクションを確定します。

+
image::exercise3/03-image-disk-012.png[title="Confirm disk detach", link=self, window=blank, width=100%]

. ディスク（`vdc`）が削除されたことを確認します。
+
VMのページから *Console（コンソール）* タブをクリックします。必要に応じて、青い *Connect（接続）* ボタンをクリックします。
+
コンソールウィンドウの上部に表示されている *User name（ユーザー名）* と *Password（パスワード）* の資格情報を使用し、*Copy to clipboard（クリップボードにコピー）* と *Paste to console（コンソールに貼り付け）* を使って仮想マシンにログインします。
+
`lsblk` コマンドを実行すると、ホットプラグされたディスクがシステム上に存在しなくなっていることが確認できます。

+
[NOTE]
====
デタッチされたディスクは PVC としてまだ存在しています。再アタッチしたり、別のVMにアタッチしたり、不要になった場合は削除したりすることができます。
====

+
image::exercise3/03-image-disk-013.png[title="Disk is removed in OS", link=self, window=blank, width=100%]

[#bonuslabinstructions]
=== Bonus Lab - Convert a Hot Plug Disk to a Permanent Disk

以下の手順は、新しいディスクのホットプラグに直接関係しないため、オプション（任意）です。ただし、ホットプラグされたディスクを標準ディスクに変換する方法（VMの再起動が必要）を知っておくと有益な場合があります。

. 上記の手順でデタッチした PVC は、手動で削除されるまで存在し続けます。したがって、VMがシャットダウンしている間に、その PVC を標準ディスクとして再アタッチすることができます。
+
. *Virtualization（仮想化） -> Virtual Machines（仮想マシン）* に移動し、*hot-plug* プロジェクトを選択します。*hot-plug-vm1* という名前のVMを選択します。
+
*四角い停止ボタン* を使用するか、*Actions（アクション）* メニューを選択して *Stop（停止）* をクリックし、VMを *停止* します。
+
image::exercise3/03-image-disk-persist-002.png[title="Stop the VM", link=self, window=blank, width=100%]

+
. *Configuration（構成）* タブをクリックし、*Storage（ストレージ）* を選択します。
+
*Add（追加）* プレダウンメニューをクリックし、*Volume（ボリューム）* を選択します。
+
image::exercise3/03-image-disk-persist-003.png[title="Add a volume", link=self, window=blank, width=100%]

. 必要に応じてディスクに名前を付け、*PersistentVolumeClaim name（PersistentVolumeClaim名）* のドロップダウンから、以前に切断した PVC（例：*dv-hot-plug-vm1-data-disk-01-um6O11*）を選択します。
+
*Save（保存）* をクリックします。
+
image::exercise3/03-image-disk-persist-004.png[title="Select the PVC", link=self, window=blank, width=100%]

+
. これで、仮想マシンの構成にブートボリュームと新しいディスクの両方が表示されます。
+
[NOTE]
====
新しいディスクは、VMの実行中に追加されていないため、*Hotplug persistent（永続ホットプラグ）* のラベルは表示されなくなります。
====

+
image::exercise3/03-image-disk-persist-005.png[title="Both disks are attached", link=self, window=blank, width=100%]
+
. *青いPlayアイコン（起動ボタン）* をクリックするか、*Actions（アクション）* メニューを選択して *Start（開始）* をクリックし、仮想マシンを起動します。
+
image::exercise3/03-image-disk-persist-006.png[title="Both disks are attached", link=self, window=blank, width=100%]

. 仮想マシンが *Running（実行中）* になったら、*Console（コンソール）* タブをクリックします。必要に応じて、青い *Connect（接続）* ボタンをクリックします。
+
コンソールウィンドウの上部に表示されている *User name（ユーザー名）* と *Password（パスワード）* の資格情報を使用し、*Copy to clipboard（クリップボードにコピー）* と *Paste to console（コンソールに貼り付け）* を使って仮想マシンにログインします。
+
`lsblk` を実行すると、名前が変更されている可能性がありますが、ディスクがアタッチされていることを確認できます。
+
ディスクがホットプラグされていたときにフォーマットを行ったりデータを書き込んだりしていた場合、データがディスク上に残っていることが確認できます。
+
image::exercise3/03-image-disk-persist-007.png[title="The disk is attached", link=self, window=blank, width=100%]

[#nic]
== Adding a NIC to a Running VM
このラボでは、既存のVMに新しい *ネットワークインターフェースカード*（NIC）を追加します。ポッドネットワーク上に単一のNICを持つ既存のVMから開始し、OpenShift Virtualizationコンソールを使用して、新しいネットワーク上の追加のNICが含まれるようにVMの構成を変更します。

[#nicinstructions]
=== Instructions

. *Webブラウザ* から <<labaccess,admin ユーザーとして OpenShift コンソールにログイン>>していることを確認し、次のステップに進みます。

+
. *OpenShiftコンソール* を使用し、*Virtualization（仮想化） → Virtual Machines（仮想マシン）* に移動します。
+
*All projects（すべてのプロジェクト）* の下で、*hot-plug* ネームスペースを選択し、*hot-plug-vm1* VMを選択します。

+
. VMが実行されている *Node（ノード）* （例：*worker-cluster-m2ssq-3*）を書き留めます。

+
image::exercise3/03-image-nic-002.png[title="Identify the worker node", link=self, window=blank, width=100%]

+
. *Configuration（構成）* タブをクリックし、*Network（ネットワーク）* を選択します。
+
VMに現在 *単一のNIC* しか存在しないことを確認します。

+
image::exercise3/03-image-nic-003a.png[title="Single NIC only", link=self, window=blank, width=100%]

+
*Console（コンソール）* タブを介してVMにログインし、オペレーティングシステムから *NIC* の詳細を確認することもできます。

+
image::exercise3/03-image-nic-003b.png[title="Single NIC in Console", link=self, window=blank, width=100%]

+
. Network（ネットワーク）の構成画面に戻り、*Add network interface（ネットワークインターフェースの追加）* ボタンをクリックします。
+
新しいウィンドウで、新しいインターフェースの名前（例：*nic-1*）を入力します。
+
モデルは *virtio*、ネットワーク（Network）はドロップダウンから *hot-plug/east-west-nad* を選択します。
+
*Save（保存）* ボタンをクリックします。

+
image::exercise3/03-image-nic-004.png[title="Add network interface", link=self, window=blank, width=100%]

+
. VM名の下に表示された *Pending changes（保留中の変更）* バナーの横にある *>* 記号をクリックします。
+
メッセージに「NICの追加を有効にするには再起動またはマイグレーションが必要である」旨が示されていること、および *Network interfaces* セクションの新しいNICに *Pending（保留中）* ラベルが付いていることを確認します。
+
image::exercise3/03-image-nic-005.png[title="View pending changes", link=self, window=blank, width=100%]

. NICが追加されると、ハードウェアの追加を完了するためにVMが *自動的にライブマイグレーション* されます。
+
NOTE: 新しいNIC構成を有効にするために、仮想マシンはあるワーカーノードから別のワーカーノードへ自動的にマイグレーション（移行）されます。新しいNIC構成がすぐに表示されない場合は、数分待つ必要があります。

+
. *Console（コンソール）* タブに戻ります。必要に応じて、青い *Connect（接続）* ボタンをクリックします。
+
コンソールウィンドウの上部に表示されている *User name（ユーザー名）* と *Password（パスワード）* の資格情報を使用し、*Copy to clipboard（クリップボードにコピー）* と *Paste to console（コンソールに貼り付け）* を使って仮想マシンにログインします。

+
. VMコンソールに新しいハードウェアが追加されたというメッセージが表示される場合があります。
+
image::exercise3/03-image-nic-008.png[title="Messages on console about new NIC", link=self, window=blank, width=100%]
+
. これで新しいNICが使用可能になります（例：環境によっては異なる名前になる場合がありますが、*enp2s0* など）。
+
オペレーティングシステムから `ip address` コマンドを実行し、出力を確認することで、新しい *NIC* を表示できます。
+
image::exercise3/03-image-nic-009.png[title="New NIC is up", link=self, window=blank, width=100%]

[IMPORTANT]
====
次のラボ用に十分なリソースを確保するため、*Terminal（ターミナル）* ウィンドウから *virtctl* を使用して *hot-plug-vm1* VM を停止してください。

[,shell,role="execute wrap",subs="attributes"]
----
virtctl stop hot-plug-vm1 -n hot-plug
----
*VM hot-plug-vm1 was scheduled to stop*
