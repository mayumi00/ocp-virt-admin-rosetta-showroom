== OpenShift Virtualization ロゼッタストーンラボへようこそ

OpenShift Virtualizationはあらゆる場所で利用されており、当社の顧客アカウント全体に大きな影響を与えています。

関心は高いものの、常に避けられない疑問があります。「他のハイパーバイザーと比較してどうなのか？」

このラボでは、OpenShift Virtualizationと他の仮想化プラットフォームを結びつける「ロゼッタストーン」のようなものを活用するプロセスをご案内します。

ロゼッタストーンは、従来の仮想化プラットフォームとOpenShift Virtualizationの機能間のマッピングを作成します。

このマッピングを実践することで、知識を飛躍的に向上させ、お客様からの質問に自信を持って対応できるようになります。

このラボでは、インフラストラクチャの最適化、ワークロードの可用性、スケーラビリティ、リソース管理のための主要な管理者機能とVMオーナー機能について、説明とデモンストレーションの方法をご紹介します。

このラボを通して、以下の内容を理解し、知識と経験を身につけ、実践できるようになります。

* *Live Migration of Workloads:* ダウンタイムなしでホスト間でVMをシームレスに移動
* *CPU and Memory Overcommit:* クラスター全体のリソース利用率を最適化する
* *Hot Plugging of CPU, Memory, and Disk Resources:* 再起動せずにVMリソースを拡張する
* *Affinity and Anti-Affinity for VM Placement:* パフォーマンスと可用性を考慮して、仮想マシンの実行場所を制御します。
* *Dynamic Rescheduling of Workloads:* プラットフォームがVMの配置を自動的に最適化します
* *Fencing and How to Handle Node Failure:* インフラストラクチャの障害発生時にワークロードの回復力を確保する
* *Decentralized Live Migration:* 名前空間間およびクラスター間でワークロードを移行する方法
* *VM Isolation and Security:* パフォーマンスと可用性を考慮して、仮想マシンの実行場所を制御します。

== VMware and OpenShift Virtualization

VMwareとOpenShift Virtualizationは機能面でほぼ同等になりつつあり、十分な知識を持つ顧客にとっては導入は容易です。

しかし、製品開発の理念が異なるため、管理者や経営陣の目にはOpenShift VirtualizationとVMwareの間には大きな違いが見られます。

VMwareはデスクトップソフトウェア企業として創業したため、すべての機能はまずグラフィカルインターフェースに実装されます。

これにより、一般ユーザーからはほとんどの複雑な機能が隠蔽されます。

VMwareの管理者や経営陣は、仮想環境の設定と管理にグラフィカルインターフェースに依存しており、非常に高度な操作はこのインターフェースの裏側に隠されています。

さらに、VMwareはプロプライエタリソフトウェア企業であるため、より広範なコミュニティとの連携は必須条件ではなく、自動化を可能にする抽象化は優先的に実装されていません。

一方、OpenShift Virtualizationは、それが属するKubernetesプロジェクト（kubevirt）と同様に、「グラフィカルインターフェース優先」のアプローチは採用していません。

Red Hatは1990年代からエンタープライズ規模の仮想化に取り組んでおり、現在も革新を続けていますが、私たちの関心はコミュニティとの連携にあり、グラフィカルな管理を最優先事項とはしていません。

== ラボ環境

右側のターミナルウィンドウは、既に踏み台ホスト経由でラボ環境に *{ssh_user}* としてログインしています。

OpenShiftコンソールが指定されていない限り、このラボのすべての手順は、踏み台ホスト上で *{ssh_user}* として OpenShift クラスタに *admin* としてログインした状態で実行してください。

ラ​​ボ環境は Red Hat OpenShift 4.20 で動作しており、以下の必要なオペレーターがプリインストールされています。

* OpenShift Virtualization
* OpenShift Data Foundations
* Node Health Check Operator
* Self Node Remediation Operator
* Kube Descheduler Operator
* Migration Toolkit for Containers
* Migration Toolkit for Virtualization
